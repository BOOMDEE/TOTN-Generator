# TOTN 生成器说明

TOTN（Traceable One-Time Number）是一种可追溯、一次性的数字编号方案，
用于防伪标识、签名编号、礼物编号、任务管理、归档冷存储等场景。

## TOTN 总体结构

TOTN 由以下字段组成（连续数字，无分隔符）：

[D][S][V][T][C][P]

字段说明：

| 字段 | 位数 | 含义 | 说明 |
|------|------|------|------|
| D    | 8    | 日期混合值 | 日期 (YYYYMMDD) + 固定码（462675）数字相加 |
| S    | 2    | 当日序号 | 01–99，当天第几个编号 |
| V    | 2    | 版本号 | 规则升级或兼容 |
| T    | 2    | 类型码 | 01=签名，02=礼物，03=文件，04=任务，05=冷存储任务 … |
| C    | 2    | 确认码 | S+V+T+P 数字总和 mod 100 |
| P    | 2    | 优先级码 | 数字越大优先级越高，0表示不适用 |

---

## 类型码（T）规划

| T值 | 含义 |
|-----|------|
| 01  | 签名 |
| 02  | 礼物 |
| 03  | 文件 |
| 04  | 任务 |
| 05  | 冷存储任务 |
| 06  | 批次 |
| 07  | 活动 |
| 08  | 特殊项目 |
| 09  | 保留 |
| 10  | 保留 |

## 优先级码（P）规划

| P值 | 含义 |
|-----|------|
| 0   | 不适用 |
| 1   | 低 |
| 2   | 中低 |
| 3   | 中 |
| 4   | 中高 |
| 5   | 高 |
| 6   | 很高 |
| 7   | 紧急 |
| 8   | 超紧急 |
| 9   | 极高 |

---

## TOTN 示例

假设输入：

- 日期：2025-12-26
- 当日序号：01
- 版本号：01
- 类型码：05（冷存储任务）
- 优先级码：6（很高）

计算：

- D = 20251226 + 462675 = 20713801
- C = (0+1 + 0+1 + 0+5 + 6) % 100 = 13

输出 TOTN：
207138010101051306

用户看到编号即可解析：

- D=20713801 → 日期混合值  
- S=01 → 第1个编号  
- V=01 → 版本  
- T=05 → 冷存储任务  
- C=13 → 校验码  
- P=06 → 很高优先级

TOTN 保持数字连续，可用于防伪、追溯及管理优先级。
